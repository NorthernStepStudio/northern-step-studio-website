2️⃣ High-Level Architecture (Critical)
┌─────────────┐
│   UI (Web)  │  ← sliders, presets, buttons
└──────┬──────┘
       │ IPC (JSON)
┌──────▼──────┐
│  Controller │  ← emotion, voice state
└──────┬──────┘
       │
┌──────▼─────────┐
│  Audio Engine  │
│  • TTS         │
│  • Emotion     │
│  • SFX Mixer   │
└──────┬─────────┘
       │
┌──────▼──────┐
│ Output Device │
└──────────────┘


Golden rule:
UI never touches audio directly.

3️⃣ IPC: UI ↔ Python (Simple + Fast)
UI → Backend message
{
  "type": "update_emotion",
  "payload": {
    "energy": 0.8,
    "calm": 0.3,
    "anger": 0.1
  }
}

Python handler
def handle(msg):
    if msg["type"] == "update_emotion":
        emotion_controller.update(msg["payload"])


Use:

stdin/stdout (simple)

or WebSocket (cleaner, scalable)

4️⃣ Audio Engine (Non-Negotiable Design)

You need 3 audio buses:

[TTS BUS] ──┐
[SFX BUS] ──┼──► MASTER ─► OUTPUT
[AMBIENT] ──┘

Ducking (Critical for polish)

When TTS speaks:

UI sounds drop −6 to −10 dB

Fade back smoothly

if tts_active:
    sfx_gain = 0.4
else:
    sfx_gain = 1.0

5️⃣ Preset System (This Makes It Usable)
Preset structure
{
  "name": "Calm Assistant",
  "voice": "female_01",
  "emotion": {
    "calm": 0.8,
    "energy": 0.2
  },
  "pitch": -1.5,
  "speed": 0.95
}

Save / Load

JSON on disk

Hot-load without restart

def load_preset(p):
    voice.set(p["voice"])
    emotion_controller.update(p["emotion"])

6️⃣ Real-Time Safety Rules (Important)

✔ Audio thread NEVER blocks
✔ TTS generation off main loop
✔ Ring buffer for playback
✔ Preload SFX at startup

If audio glitches → users will hate it.
