# Studio/Core Knowledge

This lane is the canonical source for how NStep AI explains itself, routes work, and sets boundaries across the studio.

## What this lane must answer

- What NStep AI is and is not.
- Which product should be used for a given request.
- What the assistant remembers across sessions.
- What counts as a supported capability versus a future idea.

## MVP truth set

- Positioning and portfolio overview.
- Product map and lane taxonomy.
- Global assistant scope and refusal policy.
- Shared account/session model.
- Privacy and retention summary.

## P0 missing items

- Canonical product map and definitions.
- Unified terminology glossary.
- Global scope and refusal policy.
- Memory semantics and retention rules.

## P1 missing items

- Transparency spec for lane selection and evidence display.
- Studio-wide changelog template.
- Cross-product escalation playbook.

## Retrieval hooks

- Questions about the studio should prefer this lane over product lanes.
- Broad assistant-behavior questions should cite this lane first.
- If a request is not covered here, the answer should say so explicitly.

## Safety notes

- Do not invent cross-product capabilities.
- Do not imply the assistant remembers more than the docs allow.
- Privacy and retention claims should only come from the canonical policy docs.
