# PasoScore Knowledge

This lane is the canonical source for scoring logic, event models, score interpretation, and transparency.

## What this lane must answer

- What does the score mean?
- Who is it for?
- How do score events aggregate?
- How should users interpret changes over time?

## MVP truth set

- Product intent document.
- Event model.
- Scoring logic.
- Use cases and integrations.
- Transparency notes.

## P0 missing items

- Score definition and audience.
- Event model.
- Core scoring logic.

## P1 missing items

- Interpretation guide.
- Anti-gaming rules.
- Permissions and segmentation semantics.

## P2 later items

- Trend and uncertainty metrics.
- Advanced analytics definitions.
- Calibration and fairness notes.

## Retrieval hooks

- Questions about the meaning of a score should point to the product intent doc first.
- Questions about aggregation should be grounded in the event model.

## Safety notes

- Keep fairness and transparency explicit.
- Avoid treating the score as an absolute truth if the model is heuristic.
